# 拓麻歌子 Paradise 遗传搭配模拟器

一个基于拓麻歌子 Paradise（万代 2025 年发售）育种机制的网页模拟器：输入两位父母 + 育成场 + 喂食模式，预测后代的（物种 × 面部 × 体色）组合及概率，帮玩家在游戏里花掉几小时之前先算一遍。

## 功能

- **遗传计算**：面部和体色按 50/50 在父母两位间分配；本体（与育成场同生态的父母）特征 -50%、对方 +50%（自动判定）
- **物种判定**：4 个生态（陆地 / 天空 / 海洋 / 翠林）× 17 个物种（4 主食 A + 4 主食 B + 4 主食 C + 4 杂食 + 1 隐藏）= 68 个
- **麻麻圈机制**：0→01、1→02、2-5→03、6→04 决定主食大类下出哪只
- **隐藏物种解锁**：与 ≥ 2 个不同生态联机 + 非杂食模式 → 后代必然为隐藏
- **像素画板**：身体 / 染色蒙版 / 面部三层独立 128×128 画布，铅笔 / 橡皮 / 填充 / 取色 / 移动 / 导入 PNG
- **资源全本地**：所有像素图存浏览器 IndexedDB，刷新不丢；物种 / 主食可重命名
- **收藏**：单个 (物种 + 面部 + 体色) 组合一键收藏，含完整育成上下文，可一键还原

## 运行

不需要 npm，不需要构建。但**必须经 HTTP 加载**（直接双击 `file://` 打开会因为 CORS 拒载 ES module）：

```bash
# 在项目根目录起一个静态服务器
python -m http.server 8000
# 或：npx serve
```

浏览器打开 `http://localhost:8000`。

## 数据持久化

默认所有用户数据（像素图、自定义命名、联机记录、麻麻圈、收藏等）存浏览器的 IndexedDB。换浏览器 / 换设备 / 清缓存 = 数据丢失。

**跨设备同步**：进 ⚙ 设置 → 「资源同步」→ 「选择 / 重新连接目录」选定本仓库根目录，之后所有变更会自动写入 `assets/library.json`，commit + push 即同步到 GitHub。**仅 Chrome / Edge 支持**（File System Access API）。其它浏览器只能纯本地用。

新设备克隆仓库后，按相同步骤选定本仓库根目录，会自动从 `assets/library.json` 恢复全部数据。

## 目录结构

```
.
├── index.html              # 入口
├── styles.css
├── src/
│   ├── main.js             # 启动 + 顶层 wiring
│   ├── genetics.js         # 纯遗传计算（含 autoSelfParent）
│   ├── storage.js          # IndexedDB 封装 + 内存缓存 + 订阅
│   ├── data/               # 物种 / 面部 / 体色 / 生态 / 麻麻圈 / 联机 / 收藏
│   └── ui/                 # 父母卡 / 结果面板 / 物种库 / 面部库 / 像素画板 等
└── assets/
    └── README.md           # IndexedDB schema 说明
```
